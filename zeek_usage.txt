### 1. How to Use Zeek for Proper Anomaly & Attack Identification (CTF Style)

Zeek doesn’t just “show packets” like Wireshark — it **extracts high-level logs** and **flags suspicious stuff automatically** in `notice.log` and `weird.log`.

#### Step-by-Step Workflow (Do This Every Time)

1. **Process the PCAP** (basic + full detection):
   ```bash
   # Basic (fast)
   zeek -r yourfile.pcap

   # Recommended for CTFs — loads extra detection scripts
   zeek -C -r yourfile.pcap local
   ```
   - `-C` = ignore checksum errors (common in CTF PCAPs).
   - `local` = enables built-in anomaly detection.

2. **Key Logs Zeek Creates** (your goldmine):
   | Log File       | What It Shows                          | CTF Gold Commands |
   |----------------|----------------------------------------|-------------------|
   | `notice.log`   | **Attacks & anomalies** (scans, brute-force, weird certs, etc.) | `cat notice.log \| zeek-cut ts uid id.orig_h id.resp_h msg note` |
   | `weird.log`    | Protocol violations & strange behavior | `cat weird.log \| zeek-cut ts id.orig_h id.resp_h name addl` |
   | `conn.log`     | All connections (use for anomalies)   | `zeek-cut id.orig_h id.resp_h proto service bytes duration < conn.log \| sort \| uniq -c` |
   | `http.log`     | Web requests, User-Agents, URIs       | `zeek-cut ts id.orig_h id.resp_h method host uri < http.log` |
   | `files.log`    | Extracted files (images, exes, etc.)  | `zeek-cut source mime_type filename < files.log` |
   | `dns.log`      | DNS queries (great for tunneling)     | `zeek-cut query qtype_name < dns.log` |

3. **Built-in Anomaly & Attack Detection** (no extra coding needed):
   - **Port scans** → enabled by default → shows in `notice.log` as `Scan::Port_Scan`.
   - **Brute-force** → SSH/FTP → `SSH::Bruteforcing` or `FTP::Bruteforcing`.
   - **Weird stuff** → long connections, zero-byte replies, strange TCP flags, etc.
   - **File extraction** → add this one-liner for auto-extraction:
     ```bash
     zeek -C -r yourfile.pcap local frameworks/files/extract-all-files.zeek
     ```
     Files appear in `./extract_files/`.

4. **Pro CTF Tricks for Spotting Attacks Fast**
   - **Flag hunting**:  
     ```bash
     grep -E 'flag|CTF|{' *.log
     ```
   - **Unusual traffic**:  
     ```bash
     # Long DNS queries (tunneling)
     zeek-cut query < dns.log | awk 'length($0)>100'
     # Weird User-Agents
     zeek-cut user_agent < http.log | sort | uniq -c | sort -nr
     ```
   - **Notice log magic** (your #1 place for attacks):
     ```bash
     cat notice.log | zeek-cut ts note msg id.orig_h id.resp_h | column -t
     ```
   - Enable even more detections (optional but powerful):
     ```bash
     zkg install zeek-sniffpass   # cleartext passwords
     zkg install file-extraction  # better file carving
     ```

**Quick cheat-sheet command** you can alias:
```bash
alias zeek-analyze='zeek -C -r $1 local && echo "=== NOTICE ===" && cat notice.log | zeek-cut ts note msg | column -t'
```

Run `zeek-analyze capture.pcap` and you’ll instantly see anomalies.

#### Script 1: `zeek-ctf-auto.sh` (One-command PCAP processor — your daily driver)
```bash
#!/bin/bash
# ============== EDIT THESE ==============
PCAP="yourfile.pcap"          # ← change this
OUTPUT_DIR="zeek_output"
# ========================================

mkdir -p "$OUTPUT_DIR"
cd "$OUTPUT_DIR"

echo "[+] Processing $PCAP with full detection..."
zeek -C -r "../$PCAP" local frameworks/files/extract-all-files.zeek

echo "[+] Extracting files..."
ls extract_files/ 2>/dev/null || echo "No files extracted"

echo "[+] Quick anomaly summary:"
echo "=== NOTICE (attacks/anomalies) ==="
cat notice.log 2>/dev/null | zeek-cut ts note msg id.orig_h id.resp_h | column -t | head -20

echo "=== WEIRD (protocol issues) ==="
cat weird.log 2>/dev/null | zeek-cut ts name addl id.orig_h | head -10

echo "=== FLAG HUNT ==="
grep -E 'flag|CTF|\{' *.log 2>/dev/null || echo "No obvious flags found"

echo "[+] Done! Logs in $OUTPUT_DIR"
```

Make executable: `chmod +x zeek-ctf-auto.sh`  
Usage: `./zeek-ctf-auto.sh`

#### Script 2: `flag-hunter.py` (Python flag + anomaly hunter — super easy to edit)
```python
#!/usr/bin/env python3
# ============== EDIT HERE ==============
LOG_DIR = "."          # folder with Zeek logs
FLAG_PATTERN = r'flag\{.*?\}|CTF\{.*?\}'
# =======================================

import os, re, glob

print("[+] Zeek Flag & Anomaly Hunter")

# Search all logs for flags
for log in glob.glob(f"{LOG_DIR}/*.log"):
    with open(log, 'r', errors='ignore') as f:
        for line in f:
            match = re.search(FLAG_PATTERN, line)
            if match:
                print(f"🚩 FLAG FOUND in {log}: {match.group(0)}")

# Quick anomaly stats
print("\n[+] Top Notice Types (attacks):")
if os.path.exists("notice.log"):
    with open("notice.log") as f:
        notices = [line.split()[-1] for line in f if "notice" in line.lower()]
        from collections import Counter
        print(Counter(notices).most_common(10))
```

Run with `python3 flag-hunter.py`

#### Script 3: `multi-pcap-processor.sh` (Process 10+ PCAPs at once)
```bash
#!/bin/bash
# ============== EDIT THESE ==============
PCAP_DIR="/path/to/pcaps"     # folder full of .pcap files
# ========================================

for pcap in "$PCAP_DIR"/*.pcap*; do
    echo "[+] Processing $pcap"
    filename=$(basename "$pcap")
    mkdir -p "zeek_$filename"
    cd "zeek_$filename"
    zeek -C -r "../$pcap" local
    grep -E 'flag|CTF' *.log > flags_found.txt 2>/dev/null
    cd ..
done
echo "All PCAPs processed!"
```

#### Script 4: `zeek-anomaly-stats.sh` (Quick stats for conn.log — spot weird IPs)
```bash
#!/bin/bash
# ============== EDIT THESE ==============
LOG="conn.log"
THRESHOLD=50   # connections from same IP = suspicious
# ========================================

echo "Top talkers (possible scanners/exfil):"
zeek-cut id.orig_h < "$LOG" | sort | uniq -c | sort -nr | head -15

echo -e "\nConnections with zero bytes (possible failed attacks):"
zeek-cut id.orig_h id.resp_h orig_bytes resp_bytes < "$LOG" | awk '$3==0 && $4==0' | head -10
```
